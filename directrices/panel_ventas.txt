Componentes internos:

1. SECCIÓN CLIENTE (izquierda):
   - Header: "Cliente" + botón "+ Agregar Cliente"
   - Si no hay cliente: mensaje "No hay cliente seleccionado"
   - Si hay cliente: Card con:
     - Nombre completo
     - DNI
     - Teléfono
     - Si es cuenta corriente: Saldo y límite con barra de progreso
     - Botón "Cambiar Cliente"

2. SECCIÓN PRODUCTOS (centro):
   - Header: "Productos"
   - Input de búsqueda con icono lupa
   - Lista scrolleable de productos:
     - Nombre del producto
     - Categoría (badge pequeño)
     - Precio (grande, naranja)
     - Stock disponible (gris)
     - Botón "+" azul circular
   - Lazy loading o virtualized list

3. SECCIÓN CARRITO (derecha):
   - Header: "Carrito de Compra" + contador
   - Lista de items:
     - Nombre producto
     - Precio unitario
     - Cantidad (con +/- para modificar)
     - Subtotal
     - Botón "X" para eliminar
   - Total al final (grande, bold)

4. SECCIÓN FORMA DE PAGO (debajo del carrito):
   - Select con opciones:
     - Efectivo
     - Tarjeta de Crédito/Débito
     - Transferencia
     - Cuenta Corriente (solo si cliente seleccionado tiene CC)
   - Si es Cuenta Corriente: mostrar alerta si excede límite

5. FOOTER:
   - Botón "Cancelar" (gris)
   - Botón "Finalizar Venta" (morado, grande)
     - Disabled si carrito vacío o sin cliente o sin forma de pago

Lógica:

1. Context para carrito:
   - addItem(product, quantity)
   - removeItem(productId)
   - updateQuantity(productId, quantity)
   - clearCart()
   - total

2. Al finalizar venta:
   - Validar campos obligatorios
   - POST a /api/Sale/create-sale
   - Si responde con venta normal: Mostrar éxito y cerrar
   - Si responde con ventaPendiente: Mostrar alert "Venta PENDIENTE, excede límite en $X"
   - Limpiar carrito
   - Recargar lista de ventas

3. Modal de selección de cliente:
   - Buscar clientes
   - Mostrar lista con radio buttons
   - Botón "Seleccionar"
